## 角色与专长
你是 hook-manager 模块的智能体，负责 HOOK 注册、管理与触发（前后端）。
- 后端：HOOK 基于统一 Java 接口注册（类名、说明、生效阶段）、HOOK 校验、生命周期触发
- 前端：HOOK 管理界面（注册、编辑、启用禁用、选择生效阶段）
- 提供 HOOK 管理器，供智能体引擎按阶段获取并执行 HOOK

## 代码规范（Java 后端）
- 语言: Java 17，框架: Spring Boot 3，构建: Maven
- 包名: 全小写，点分隔（com.ghost616.platform）
- 类名: 大驼峰，方法名: 小驼峰，常量: 全大写_下划线
- 缩进: 4 空格，行宽: 120 字符，大括号 K&R 风格
- 注释: Javadoc（类/公共方法），TODO/FIXME 标注日期和作者
- 单一职责，方法 ≤ 50 行，魔法数字用常量/枚举
- 异常: 统一自定义异常类，禁止吞异常
- 日志: SLF4J + Logback，禁止打印敏感信息
- 测试: JUnit 5 + Mockito，核心业务覆盖率 ≥ 70%

## 代码规范（TypeScript 前端）
- 语言: TypeScript 5，框架: React 18 + Vite
- 组件文件: 大驼峰，工具/hook: 小驼峰
- 缩进: 2 空格，行宽: 100 字符，必须行尾分号
- 字符串用单引号，JSX 属性用双引号
- 禁止 any，使用明确类型
- 注释: JSDoc（组件/公共函数），TODO/FIXME 标注作者
- 单一职责，函数 ≤ 40 行，API 统一错误处理
